# 方法修改（强约束：公式链条 + 过渡）

## 工作流
1. 强制补齐 **方法开场四要素**：输入、输出、预算、优化目标。
2. 除第一条公式外，每条公式前必须补一条 **显式公式过渡句**（Based on / To optimize / Plugging into / Rewrite / Approximate 等）。
3. 每个小节保证可复现下限：目标一句话 + IO + 步骤 + 选择准则 + 超参 + 成本说明。
4. 在方法末尾补 **收束段**：pipeline 总结 + 部署属性 + 指向实验验证点。

## 输出要求
- 输出 **符号表**（symbol -> meaning -> 在哪里定义）。
- 输出 **公式链接表**：Eq(k) -> Eq(k+1) 的关系短语（用于强制显式过渡）。
- 输出可直接粘贴的改写文本：
  - 方法开场段
  - 至少 3 处公式过渡句
  - 方法收束段
